# 仕様駆動開発

- このプロジェクトでは仕様書駆動開発（SDD）を行う。

## ディレクトリ構成

- `specs/`: 仕様書
- `src/`: 実装
- `tests/`: 受け入れテスト（仕様書と対応するファイル名にする）

## 開発フロー

- 実装前に必ず仕様書を読む。
- 仕様書にあいまいな点があればユーザーに質問し仕様書を更新する。
- 他の仕様書との整合性を確認し、矛盾があればユーザーに確認してから仕様書を更新する。
- 仕様書から受け入れテストを定義し、それを満たすように実装する。

# Gitワークフロー

- 必ず最新の `main` からworktreeを作成して開発する。
- 同一の変更理由を持つ変更を1つのコミットに含める。
- 変更が完了したらpushしてPull Requestを作成する。 `main` に直接mergeしてはいけない。
- 作業の区切りごとに `gh pr view` でマージ済みのPRを確認し、対応するworktreeとブランチを削除する。

# 技術方針

## 技術スタック

- 言語: TypeScript
- ランタイム: Node.js
- パッケージ管理: pnpm
- ツールキット: Vite+
- Web UI: React Router（メタフレームワークとして使用し、APIサーバーも兼ねる）

## アーキテクチャ

- エージェント向けのCLIと人間向けのWeb UIを提供し、タスクの閲覧や操作をおこなう。
- タスクごとにfrontmatter付きのmarkdownファイルを保存する。
- タスク間の関係性をJSONファイルに保存する。
