# ディレクトリ構成

- `specs/`: 仕様書
- `src/`: 実装
- `tests/`: 受け入れテスト（仕様書と対応するファイル名にする）

# 技術方針

## 技術スタック

- 言語: TypeScript
- ランタイム: bun
- パッケージ管理: bun
- Web UI: React + TanStack Router

## アーキテクチャ

- エージェント向けのCLIと人間向けのWeb UIを提供し、タスクの閲覧や操作をおこなう。
- タスクごとにfrontmatter付きのmarkdownファイルを保存する。
- タスク間の関係性をJSONファイルに保存する。

## Web UI の画面設計方針

- FigmaなどのデザインツールではなくClaudeによる高速プロトタイピングで進める。
- 動くものを作りながら要件を固めていく。
- 仕様書は実装後に振り返りとして書くか、必要に応じて省略する。

# テストの実行

- `bun test` でビルドなしにテストを実行できる。
- `dist/` ビルド成果物がなくてもすべてのテストがパスする状態を維持すること。
- サーバー実装で静的アセットが必要な場合は、ファイルの存在チェックをしてフォールバックを用意すること。

# 作業ログの書き方

タスクファイル（`~/.tasks/<id>.md`）のfrontmatterの直下に `# 作業ログ` セクションを作成し、今日の日付のサブセクションを追加する。

```markdown
# 作業ログ

## 2026-03-23

- 仕様書を読んで実装方針を決めた
```

- 日付ごとに `## YYYY-MM-DD` のサブセクションを作成する
- 1つの作業につき1行、箇条書きで追記する
