# インデックスファイル仕様

## 概要

タスクの優先順位、親子関係、依存関係をJSONファイルとして保存する。

## 保存場所

`~/.tasks/index.json`

## フォーマット

以下のフィールドを持つJSONオブジェクト。

| フィールド     | 型     | 説明                                                                                                    |
| -------------- | ------ | ------------------------------------------------------------------------------------------------------- |
| `children`     | object | タスクIDをキー、子タスクIDの順序付き配列を値とするオブジェクト。`"root"` キーがルートレベルのタスク順序 |
| `dependencies` | object | タスクIDをキー、依存するタスクIDの配列を値とするオブジェクト                                            |

```json
{
  "children": {
    "root": [3, 1, 2],
    "1": [4, 5]
  },
  "dependencies": {
    "1": [2, 3]
  }
}
```

上記の例では：

- タスク3、1、2がルートレベルのタスク（この順で優先度が高い）
- タスク1はタスク4とタスク5を子に持つ（タスク4の方が優先度が高い）
- タスク1はタスク2とタスク3に依存している（タスク2と3が完了するまでタスク1を開始できない）

## デフォルト値

ファイルが存在しない場合は以下の値として扱う。

```json
{
  "children": {
    "root": []
  },
  "dependencies": {}
}
```
