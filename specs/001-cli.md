# CLI仕様

## 概要

`task` コマンドはエージェントと人間がタスクを管理するためのCLIツール。

## コマンド名

`task`

## グローバルオプション

- `--version`, `-v`: バージョンのみを表示する（例: `0.1.0`）
- `--help`, `-h`: ヘルプを表示する

## 出力形式

すべてのコマンドの出力はJSON形式で標準出力に出力する。

### 正常系

exit codeは `0`。

```json
{ "ok": true, "result": {} }
```

`result` の内容はコマンドによって異なる。

### 異常系

exit codeは `1`。

```json
{ "ok": false, "error": { "message": "...", "usage": "...", "retriable": false } }
```

| フィールド           | 型                           | 説明                                                                                                                              |
| -------------------- | ---------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| `error.message`      | string                       | エラーの説明。エージェントが次の行動を判断できる内容にする                                                                        |
| `error.usage`        | string \| null               | 正しいコマンドの使い方。使い方の問題でない場合は `null`。エージェントが修正方法を判断できるようにする                             |
| `error.retriable`    | boolean                      | `true` のとき、同じコマンドをそのまま再試行してよい（一時的なエラー）。`false` のとき、エージェントは引数などを修正する必要がある |
| `error.subcommands`  | array \| null                | サブコマンドが必要なコマンドで、サブコマンドが指定されなかった場合に利用可能なサブコマンドの一覧を返す。それ以外は `null`         |

`error.subcommands` の各要素は以下のフィールドを持つ。

| フィールド  | 型     | 説明                 |
| ----------- | ------ | -------------------- |
| `name`      | string | サブコマンド名       |
| `description` | string | サブコマンドの説明 |

```json
{
  "ok": false,
  "error": {
    "message": "subcommand is required",
    "usage": "task <command> <subcommand>",
    "retriable": false,
    "subcommands": [
      { "name": "add", "description": "..." },
      { "name": "delete", "description": "..." }
    ]
  }
}
```
