# `add` コマンド仕様

## 概要

新しいタスクを作成するコマンド。

## 書式

```
task add <title> [--parent <id>]
```

## 引数

| 引数/オプション | 必須 | 説明                                             |
| --------------- | ---- | ------------------------------------------------ |
| `<title>`       | 必須 | タスクのタイトル                                 |
| `--parent <id>` | 任意 | 親タスクのID。指定するとそのサブタスクとして作成 |

## 動作

1. `<title>` が指定されていない場合はエラーを出力して終了する。
2. `~/.tasks/` ディレクトリが存在しない場合は作成する。
3. `~/.tasks/` ディレクトリ内の全 `.md` ファイルのIDの最大値に1を加えた値を新しいIDとする。ファイルが存在しない場合はIDを `1` とする。アーカイブ済みタスクのファイルも含めてスキャンするため、IDは再利用されない。
4. タスクファイル仕様（`002-task-file.md`）に従い、`~/.tasks/<ID>.md` を作成する。
5. インデックスファイルに新しいIDを追加して保存する。
   - `--parent <id>` が指定された場合: 親タスクの存在確認後、`children[id]` の末尾に追加する。
   - `--parent` が指定されない場合: `children["root"]` の末尾に追加する。
6. 作成したタスクのIDを出力する。

## 出力

出力形式は共通仕様（`001-cli.md`）に従う。

### 正常系

`result.id` に作成したタスクのIDを含む。

```json
{ "ok": true, "result": { "id": 1 } }
```

### 異常系

| 条件                                | `error.message`       | `error.usage`      | `error.retriable` |
| ----------------------------------- | --------------------- | ------------------ | ----------------- |
| `<title>` が指定されていない        | `title is required`   | `task add <title>` | `false`           |
| `--parent` で指定したIDが存在しない | `Task <id> not found` | `null`             | `false`           |

## 使用例

```
$ task add "買い物をする"
{ "ok": true, "result": { "id": 1 } }
```

```
$ task add "サブタスク" --parent 1
{ "ok": true, "result": { "id": 2 } }
```

```
$ task add
{ "ok": false, "error": { "message": "title is required", "usage": "task add <title>", "retriable": false } }
```
